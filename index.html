Usage definitions
Encounter client — had at least 1 consult-complete event in the 30-day window (i.e. a billing encounter was completed in HbC)
Feature-only client — logged into HbC and used features but did not complete any consult-complete events
No activity — no events recorded in the 30-day window (client may still be contracted)
Power Clinical User — active on ≥18 days within the 30-day period with >200 total encounters
Standard Clinical User — encounter user, active but below the power threshold (>50 encounters)
Admin / Power Support — heavy feature usage (>300 feature events) with minimal or no encounters
Casual User — low engagement — few encounters and few feature events in the window
Days active — number of distinct calendar days on which any usage event was recorded for that practice in the 30-day window
AI events — events from category "5. AI Features" (Nora/Clinical Assistant usage)
Feature events — all events excluding Noise / Passive category (passive background events are excluded from meaningful usage counts)

Data architecture & refresh strategy

How this dashboard connects to your Azure pipeline and how to keep it current

📊 Data pipeline

  • Azure App Insights → Azure Logic App
  • Daily CSV export (7-day chunks)
  • Grain: User · Day · Feature
  • ~300k rows/month
  • Google Drive receives dated CSV daily

🔄 GitHub Actions refresh

  • Nightly cron at 02:00 SAST
  • Service account downloads latest CSV
  • Python aggregates → agg.json
  • Auto-commits → GitHub Pages deploys
  • BC ownership via Sheets API v4

🔒 Production security

  • Azure Static Web Apps + Azure AD
  • Uses existing HbC Microsoft accounts
  • Alt: Cloudflare Pages + Cloudflare Access
  • Free tier ≤50 users
  • Never use GitHub Pages for PHI
